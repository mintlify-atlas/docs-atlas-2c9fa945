---
title: "Get started with Komac"
description: "Learn how to install Komac, configure a GitHub token, and submit your first WinGet package manifest pull request in under five minutes."
---

Komac automates the full WinGet manifest workflow: it downloads your installers, analyzes them, generates the required YAML files, and opens a pull request to `microsoft/winget-pkgs` — all from a single command. This guide walks you through everything you need to go from zero to a submitted PR.

<Steps>
  <Step title="Install Komac">
    Install Komac using your preferred method. WinGet is the quickest option on Windows; Cargo works on every platform.

    <CodeGroup>

    ```bash WinGet (Windows)
    winget install komac
    ```

    ```bash Homebrew (macOS)
    brew install komac
    ```

    ```bash Cargo (all platforms)
    cargo install --locked komac
    ```

    </CodeGroup>

    For `.deb`, `.rpm`, and portable binary options, see the [installation guide](/installation).

    Verify the installation:

    ```bash
    komac --version
    ```
  </Step>
  <Step title="Get a GitHub token">
    Komac submits pull requests to `microsoft/winget-pkgs` on your behalf, so it needs a GitHub personal access token.

    1. Go to [github.com/settings/tokens](https://github.com/settings/tokens) and click **Generate new token (classic)**.
    2. Give the token a descriptive name, such as `komac`.
    3. Under **Scopes**, check **public_repo**.
    4. Click **Generate token** and copy the value — you won't be able to see it again.

    <Note>
      Komac requires a classic token. Fine-grained tokens can create commits and branches, but cannot open pull requests to `microsoft/winget-pkgs` due to a current GitHub limitation. See [issue #310](https://github.com/russellbanks/Komac/issues/310) for updates.
    </Note>
  </Step>
  <Step title="Store your token">
    Save the token to your system's secure credential store so you don't need to pass it on every command:

    ```bash
    komac token update
    ```

    Komac will prompt you to paste your token. It is stored using the native keychain on each platform (Windows Credential Manager, macOS Keychain, or the Linux Secret Service).

    If you prefer not to store a token, you can pass it inline with `--token` or set the `GITHUB_TOKEN` environment variable instead.
  </Step>
  <Step title="Create your first manifest">
    Choose the right command based on whether the package already exists in `microsoft/winget-pkgs`.

    **Adding a new package**

    Use `komac new` to create a package that does not yet exist in winget-pkgs. Komac guides you through every required field interactively:

    ```bash
    komac new
    ```

    You can also pass values directly to skip prompts:

    ```bash
    komac new Publisher.Package \
      --version 1.0.0 \
      --urls https://example.com/installer.exe \
      --submit
    ```

    **Updating an existing package**

    Use `komac update` to add a new version to a package that is already in winget-pkgs. Komac fetches the existing manifests and merges the new version on top:

    ```bash
    komac update Publisher.Package \
      --version 1.2.3 \
      --urls https://example.com/installer.exe \
      --submit
    ```

    For multiple installer architectures, list all URLs separated by spaces:

    ```bash
    komac update Publisher.Package \
      --version 1.2.3 \
      --urls https://example.com/installer-x64.exe https://example.com/installer-x86.exe \
      --submit
    ```

    <Tip>
      Add `--dry-run` to any command to run the full analysis and generate the manifest files locally without submitting a pull request. This is useful for checking the output before going live.
    </Tip>
  </Step>
  <Step title="Review and submit">
    If you run without `--submit`, Komac shows you the generated manifests and asks whether to open a pull request. Review the output, confirm, and Komac handles the rest:

    - Forks `microsoft/winget-pkgs` to your account if you don't already have a fork
    - Creates a branch named after the package and version
    - Pushes the manifest files
    - Opens a pull request with the correct title and body

    Once the PR is open, the WinGet validation bot runs automated checks. Most valid submissions are merged within a few days.
  </Step>
</Steps>

## Using GITHUB_TOKEN instead of a stored token

If you are running Komac in a CI environment or prefer not to use the system keychain, set the `GITHUB_TOKEN` environment variable and Komac will use it automatically:

```bash
export GITHUB_TOKEN=ghp_...
komac update Publisher.Package --version 1.2.3 --urls https://example.com/installer.exe --submit
```

The `--token` flag is also accepted on every command for one-off use.
