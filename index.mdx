Komac is a fast, intelligent CLI tool for creating and submitting [WinGet](https://github.com/microsoft/winget-pkgs) package manifests. It analyzes installers deeply — extracting metadata from Inno Setup, NSIS, MSI, MSIX, Burn, and ZIP packages — and automates the entire pull request workflow to `microsoft/winget-pkgs`.

## Why Komac?

Komac goes further than other manifest tools by performing deep installer analysis — reading Inno Setup scripts, MSI property tables, MSIX manifests, and more — to automatically fill in fields that other tools miss or require you to enter manually.

## Get started

<Steps>
  <Step title="Install Komac">
    Install via WinGet, Homebrew, Scoop, or Cargo depending on your platform.

    ```bash
    winget install komac
    ```
  </Step>
  <Step title="Add your GitHub token">
    Komac needs a GitHub classic token with the `public_repo` scope to submit pull requests.

    ```bash
    komac token update
    ```
  </Step>
  <Step title="Update a package">
    Point Komac at your installer URLs and it handles the rest — analysis, manifest generation, and PR submission.

    ```bash
    komac update Publisher.Package --version 1.2.3 \
      --urls https://example.com/installer.exe \
      --submit
    ```
  </Step>
</Steps>
